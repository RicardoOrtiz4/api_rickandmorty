Entregables:
Pantalla con datos y estados (cargando/error/vacío).
.env + lectura con dotenv/ENVied; README con pasos. Dart packages
Evidencia de pruebas en emulador y dispositivo.
Criterios técnicos: HTTPS, timeouts, validación de entrada (ciudad), sanitización de texto. OWASP Cheat Sheet
Dependencias: http, flutter_dotenv o ENVied. docs.flutter.dev, Dark Packages
Configurar API key en .env y no commitearla.
Petición GET con http, parseo JSON y manejo de errores (timeouts, 429, etc.).
